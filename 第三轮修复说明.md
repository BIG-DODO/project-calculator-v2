## P0问题1：振荡检测字段访问风险（运行时错误）

### 问题描述
`checkDistribution()` 在不需要优化时返回 `{needsOptimization: false}`，没有 `type/a/b/eff` 字段。当 `detectOscillation()=true` 但 `distCheck.needsOptimization=false` 时，访问 `distCheck.type` 等字段会得到 undefined，导致 `manualDistribute(undefined, ...)` 运行错误。

### 修复方案（采用方案B）

**修改位置**：5.2节 `checkDistribution` 函数

**修复内容**：
1. 当 `needsOptimization=true` 时，正常返回完整字段（不变）
2. 当 `needsOptimization=false` 时，**仍然返回主要类型的信息**：
   - 遍历所有类型，找到栋数最多的主要类型（dominantType）
   - 返回该类型的 `type/a/b/eff` 字段
   - 如果没有活跃配置，返回 `{needsOptimization: false, type: null, a: null, b: null, eff: null}`

**修复后代码**：
```javascript
function checkDistribution(counts, configs):
   // ... 原有逻辑 ...
   
   // 不需要优化时，返回主要类型信息（用于振荡检测场景）
   dominantType = null
   dominantCount = 0
   for each type in ['light-steel', 'split', 'layer']:
      if typeGroups[type].totalCount > dominantCount:
         dominantCount = typeGroups[type].totalCount
         dominantType = type
   
   if dominantType and typeGroups[dominantType].indices.length > 0:
      dIndices = typeGroups[dominantType].indices
      return {
         needsOptimization: false,
         type: dominantType,
         a: min(dIndices.map(i => configs[i].base)),
         b: max(dIndices.map(i => configs[i].base)),
         eff: configs[dIndices[0]].unitCap / configs[dIndices[0]].base
      }
   
   return { needsOptimization: false, type: null, a: null, b: null, eff: null }
```

## P1问题6：targetBaseLimit未考虑固定产品基底（待讨论）

### 问题描述

remainingTargetBaseBh = targetBaseBh - fixedProductsBase * (targetBaseBh / remainingBase)
