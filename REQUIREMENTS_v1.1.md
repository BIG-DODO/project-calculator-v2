## 1. 项目背景

本项目是一个基于Web的规划指标计算工具，根据用户输入的用地条件（用地面积、容积率、限高、配套占比等），自动计算各类厂房和配套建筑的产品配置。

## 2. 目标

根据用户输入的规划用地指标，自动生成产品配置表，包括：
- 各类厂房（轻钢、分栋、分层）的栋数、面积段、层数
- 配套建筑（产业大厦、配套楼、配套宿舍）的配置
- 容积率、建筑密度等指标的校验

**核心要求**：
- 容积率偏差 ≤ 0.5%（理想 ≤ 0.01%）
- 建筑密度偏差 ≤ 5%
- 面积段分布均衡（避免某面积段占比超过其他总和的2倍）

## 3. 全局变量定义

| 变量 | 含义 | 计算方式 |
|------|------|---------|
| S | 规划用地面积 | 用户输入 |
| F | 容积率 | 用户输入 |
| D | 建筑密度 | F<1.5→45%, 1.5≤F<2.0→42%, F≥2.0→40% |
| R1 | 科研办公用房占比 | 用户输入 |
| R2 | 配套用房占比 | 用户输入 |
| Hl | 限高 | 用户输入 |
| targetBase | 目标基底面积 | S × D |
| targetCap | 目标计容面积 | S × F |
| targetRdArea | 研发办公目标计容面积 | targetCap × R1 |
| targetAncillaryArea | 配套用房目标计容面积 | targetCap × R2 |
| totalAncillary | 配套用房目标计容面积（别名） | 同 targetAncillaryArea |
| exactBase | 精确基底面积（取整前） | 目标面积 / 层数 |
| maxLayerHeight | 分层厂房较高高度 | 从已选分层计算 |
| minLayerHeight | 分层厂房较低高度 | 从已选分层计算 |

**术语定义**：
- **Bh** = 较高层数选项（如分栋选2层和4层时，Bh=4）
- **Bl** = 较低层数选项（同上例，Bl=2）
- **N** = 层数（用于高度公式，区别于容积率F）
- **火车头** = 轻钢厂房首层局部加高部分，用于办公或展示，计容时额外计算
- **双拼** = 两栋厂房合并建造，共享墙体，基底面积翻倍
- **FL预处理** = 将分层/分栋厂房各面积段除以2，以减少层数类型多样性
- **强排** = 强制性规划指标排列，即在满足规划条件的前提下进行产品配置

## 5. 关键规则

### 5.1 分层厂房限高处理
- 当分层厂房高度超过限高时，**自动逐层减1**
- 允许降到任意层数（无6层下限）
- ≥5层：归类为分层，备注"因限高原因，分层降低层数"
- ≤4层：归类为分栋，面积段与原分栋并集（取消双拼，还原原始值）

### 5.2 配套用房规则

**UI选择规则**：
- `totalAncillary ≤ 2400`：用户可任意选择（仅配套楼/仅配套宿舍/两者都选）
- `totalAncillary > 2400`：强制必须选配套宿舍，配套楼可选（不强制）

**配套楼面积边界**：
| 总面积 | 配套楼面积 | 配套宿舍 | 配套楼层数 |
|--------|-----------|---------|-----------|
| < 1200 | 全部（可<1200） | 无 | 2层 |
| 1200-2400 | 全部 | 无 | 3层 |
| 2400-3600 | 1200（固定） | 剩余 | 3层 |
| 3600+ | 2400（上限） | 剩余 | 3层 |

**配套宿舍**：
- 基底取整：`floor(exactBase / 20) * 20`
- 高度超限时两次调整：标准层高3.6m → 降低至3.3m → 仍超限则用上限反推层数

### 5.3 Bh/Bl取消规则
- 平均效率 ≥ Bh效率 → 取消Bl，只用Bh
- 平均效率 ≤ Bl效率 → 取消Bh，只用Bl

### 5.4 单一层数规则
- 只用容积率控制，不考虑密度（密度自然形成）
- 丰富面积段后仍不满足时，可用手动分配算法

### 5.5 FL预处理
- 分层厂房公式 /2
- 可循环执行直到层数类型 ≤ 2
- 优先处理分层，其次分栋

### 5.6 丰富面积段
- 触发条件：遍历算法后最优解仍无法满足容积率允许误差
- 增加上限（按**用地面积S**判断）：≤20000m²→+1，≤50000m²→+2，>50000m²→+3
- 方法：递减（x-50, x+50, x-100）+ 内插（相邻两值的中间值）
- 取整：所有新增值必须是50的倍数
- 去重：新增值不能与已有值重复
- 分层新增面积段 ≤ 800 强制双拼

### 5.7 3.5层处理
- 3.5 vs 3 → 3.5是Bh
- 3.5 vs 4 → 3.5是Bl

### 5.8 轻钢计容
- 等效层数：2.1
- 独栋：基底×2+200
- 双拼：总基底×2+400

### 5.9 双拼转换规则（最终输出时执行）

**核心原则**：算法推导全程按独栋计算（分层≤800强制双拼除外），最终输出时才做双拼转换。

**转换规则**：
- 某产品某层数某面积段的栋数为**偶数** → 全部转为双拼（双拼栋数 = 独栋栋数 ÷ 2）
- 某产品某层数某面积段的栋数为**奇数** → 双拼栋数 = floor(独栋栋数 ÷ 2)，余1栋为独栋

**例外**：分层厂房中面积段≤800的类型已强制双拼（基底已×2），跳过此转换。

**示例**：
- 轻钢厂房1000面积段，算法输出6栋 → 最终输出：3栋双拼
- 分栋厂房540面积段3.5层，算法输出5栋 → 最终输出：2栋双拼 + 1栋独栋
- 分层厂房600面积段8层，算法输出4栋 → 已强制双拼，最终输出：4栋（基底已是1200）

### 5.10 分层双拼
- 600/800强制双拼
- 1000/1200不强制
- 丰富面积段后 ≤ 800 也强制双拼

### 5.10 三种厂房类型
- 强制分配1栋最低层数最小面积分层
- 剩余退化为分栋+轻钢

### 7.1 期望输出验证标准

每个场景的算法输出必须同时满足以下三个条件：

**1. 容积率偏差**
- 允许误差：≤ 0.5%
- 理想误差：≤ 0.01%
- 计算公式：`|实际计容面积 - 目标计容面积| / 目标计容面积`

**2. 建筑密度偏差**
- 允许误差：≤ 5%
- 计算公式：`|实际基底面积 - 目标基底面积| / 目标基底面积`

**3. 栋数分布均衡性**
- 同一厂房类型中，当该类型的总栋数 ≥ 5 时：
  - 任意单一面积段的栋数 ≤ 其他面积段栋数总和 × 2
- 若触发此条件，需通过丰富面积段或手动分配算法优化

## 9. 异常处理

### 9.1 输入参数异常
- `landArea <= 0`：报错"用地面积必须大于0"
- `far <= 0`：报错"容积率必须大于0"
- `heightLimit <= 0`：报错"限高必须大于0"
- `ancillaryRatio < 0` 或 `rdRatio < 0`：报错"占比不能为负"
- `ancillaryRatio + rdRatio > 1`：报错"配套占比+科研占比不能超过1"

### 9.2 产品配置异常
- 所有面积段都不满足容积率：触发丰富面积段，仍不满足则用手动分配
- 密度超限（>5%）：回退到上一轮的counts，或放宽密度约束
- 某层数因限高完全不可用：记录原因，从selectedProducts中移除

### 9.3 兜底策略
- 当所有算法都无法满足要求时，输出最接近的可行解，并标注偏差
- 提供手动调整入口，允许用户手动修改配置
