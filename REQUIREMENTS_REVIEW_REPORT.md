#### 问题S1: 变量F存在致命命名冲突 — 同时表示"容积率"和"层数"

| 项目 | 内容 |
|------|------|
| **位置** | 第3节"全局变量定义" + 第4节"产品类型基础参数"高度公式列 |
| **问题描述** | 第3节明确定义F为"容积率"（如1.0, 1.2, 2.0等）。但第4节所有产品的高度公式中F表示的是"层数"（如产业大厦公式`6.6+(F-2)*4.5+4.5+1.2`，代入F=5层得25.8m，代入F=3.0容积率得16.8m，后者显然不合理）。 |
| **影响分析** | **极高**。开发者可能在代码中错误地将容积率F代入高度公式，导致所有建筑高度计算错误。这是一个必然导致Bug的设计缺陷。 |
| **优化建议** | 将高度公式中的F改为N（或Layer、FloorCount等），明确区分：F=容积率，N=层数。修正所有高度公式：`6.6+(N-2)*4.5+4.5+1.2`等。 |

**验证计算**:
- 产业大厦5层：`6.6+(5-2)*4.5+4.5+1.2 = 25.8m` ✓
- 若误用F=3.0容积率：`6.6+(3-2)*4.5+4.5+1.2 = 16.8m` ✗（5层楼不可能只有16.8m高）

#### 问题S2: 配套楼高度写死18.3m与2层特殊情况矛盾

| 项目 | 内容 |
|------|------|
| **位置** | 第4节"产品类型基础参数"配套楼行 + 第5.2节配套楼规则 |
| **问题描述** | 第4节配套楼高度公式写死为`7.2+5.1+4.5+1.2=18.3`，对应3层高度。但同一行又注明"固定3层（特殊情况2层）"。5.2节表格显示`<1200`时配套楼为2层。2层时高度不可能仍是18.3m。 |
| **影响分析** | **高**。2层配套楼的高度未定义，开发者无法正确实现。建筑密度和容积率校验会因高度错误而产生偏差。 |
| **优化建议** | 明确2层配套楼的高度公式。合理推算：2层 = 首层7.2 + 二层5.1 + 屋面1.2 = **13.5m**（二层按非顶层的5.1m计）。在表格中补充：2层时高度=13.5m，3层时高度=18.3m。 |

#### 问题S3: 5.6节丰富面积段条件中"面积"指代不明

| 项目 | 内容 |
|------|------|
| **位置** | 第5.6节"丰富面积段" |
| **问题描述** | 规则写"增加上限：≤20000m²→+1，≤50000m²→+2，>50000m²→+3"，但未指明"面积"具体是哪个变量。可能是用地面积S，也可能是目标计容面积targetCap，或其他面积指标。 |
| **影响分析** | **高**。不同理解导致完全不同的结果。例如场景7（S=18900，targetCap=56700），按S判为≤20000→+1，按targetCap判为>50000→+3，差距达3倍。 |
| **优化建议** | 明确面积指代变量。建议补充说明如："此处面积指**用地面积S**"或"此处面积指**目标计容面积targetCap**"。 |

**数值验证**:

| 场景 | S | targetCap | 若按S判断 | 若按targetCap判断 |
|------|---|-----------|----------|------------------|
| 7 | 18900 | 56700 | ≤20000→+1 | >50000→+3 |
| 4 | 42500 | 85000 | ≤50000→+2 | >50000→+3 |
| 8 | 48000 | 144000 | ≤50000→+2 | >50000→+3 |

| 项目 | 内容 |
