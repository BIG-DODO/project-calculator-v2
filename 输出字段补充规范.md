# 新算法输出字段补充规范

`product-config-algorithm-v2.js` 的 `calculateProductConfig` 需要返回与 `product-config-v2.html` 兼容的数据结构。

## 顶层字段

| 字段 | 说明 | 计算方式 |
|------|------|---------|
| `totalBase` | 实际总基底面积 | sum(p.base * p.count) |
| `totalArea` | 实际总建筑面积/地上建筑面积 | sum(p.unitArea * p.count) |
| `totalCap` | 实际总计容面积 | sum(p.unitCap * p.count) |
| `products` | 产品配置数组 | - |

## 每个 product 字段

| 字段 | 说明 | 来源/计算 |
|------|------|----------|
| `id` | 产品标识 | 'tower' / 'dorm' / 'support' / 'light-steel' / 'split' / 'layer' |
| `type` | 产品类型名称 | '产业大厦' / '配套宿舍' / '配套楼' / '轻钢厂房' / '分栋厂房' / '分层厂房' |
| `productType` | 显示字符串 | 如 `7F(1000)`，由 floors + 'F(' + base + ')' 组成 |
| `base` | 单栋基底面积 | 算法输出 |
| `unitArea` | 单栋建筑面积 | 本版本等于 `unitCap`（全程独栋） |
| `unitCap` | 单栋计容面积 | 算法输出 |
| `count` | 栋数 | 算法输出 |
| `floors` | 层数 | 算法输出 |
| `totalHeight` | 建筑高度（含女儿墙） | 算法输出 |
| `standardHeight` | 产业大厦最终标准层层高 | 仅产业大厦，4.5 或 4.2 |
| `totalBase` | 该产品总基底 | base * count |
| `totalArea` | 该产品总建筑面积 | unitArea * count |
| `totalCap` | 该产品总计容 | unitCap * count |
| `form` | 建筑形式 | 本版本固定为 '独栋' |
| `load` | 荷载说明 | 按产品类型固定 |
| `elevator` | 电梯配置 | 按产品类型、单元整栋面积计算；`0客`/`0货` 不显示；双拼/三拼按规则追加 `分单元` 前缀或 `/` 连接 |
| `fl` | 各层层高 | { first, second, standard, top } |
| `flSuffix` | 层高后缀 | { second, standard, top }，轻钢为 '（夹）'，其他为空 |

## 各产品详细字段

### 产业大厦
- `form`: '分层分户'
- `load`: '首层2000kg，标准层350kg'
- `elevator`: `paxLift = max(1, ceil(unitCap / 4000) - 2)`，输出 `paxLift + '客2货'`
- `fl`: { first: 6.6, second: standardHeight, standard: standardHeight, top: standardHeight }
- `flSuffix`: { second: '', standard: '', top: '' }

### 配套宿舍
- `form`: '分户'
- `load`: '标准层350kg'
- `elevator`: `paxLift = ceil(unitCap / 5000)`，输出 `paxLift + '客'`（不显示 `0货`）
- `fl`: { first: 4.8, second: 3.6, standard: 3.6, top: 3.6 }
- `flSuffix`: { second: '', standard: '', top: '' }

### 配套楼
- `form`: '-'
- `load`: '首层2000kg，二层800kg，二层及以上500kg'
- `elevator`（按单栋基底 `base`）：
  - `base < 1500`：`1客`
  - `1500 <= base < 4000`：`1客1货`
  - `base >= 4000`：`1客2货`
- `fl`:
  - 2层: { first: 7.2, second: 4.5, standard: 4.5, top: 4.5 }
  - 3层: { first: 7.2, second: 5.1, standard: 4.5, top: 4.5 }
- `flSuffix`: { second: '', standard: '', top: '' }

### 轻钢厂房
- `form`: '独栋' / '双拼'（最终输出阶段转换）
- `load`: '首层2000kg'
- `elevator`: '无'
- `fl`: { first: 12, second: 4.8, standard: 3.6, top: 3.6 }
- `flSuffix`: { second: '（夹）', standard: '（夹）', top: '（夹）' }

### 分栋厂房
- `form`: '独栋' / '双拼' / '三拼'（最终输出阶段转换）
- `load`: '首层2000kg，二层800kg，二层及以上500kg'
- `elevator`（按单栋/组合后 **单元整栋面积** `unitArea = base × floors`）：
  - `unitArea < 1500`：`1货`
  - `1500 <= unitArea < 4000`：`1客1货`
  - `unitArea >= 4000`：`1客2货`
- `fl`: { first: 7.2, second: 5.1, standard: 4.5, top: 4.5 }
- `flSuffix`: { second: '', standard: '', top: '' }

### 分层厂房
- `form`: '分层分户' / '双拼' / '三拼'（最终输出阶段转换）
- `load`: '首层2000kg，二层800kg，二层及以上500kg'
- `elevator`（按单栋/组合后 **单元整栋面积** `unitArea = base × floors`）：
  - `unitArea <= 4000`：`1客1货`
  - `4000 < unitArea <= 6000`：`1客2货`
  - `unitArea > 6000`：`2客2货`
- `fl`: { first: 7.2, second: 5.1, standard: 4.5, top: 4.5 }
- `flSuffix`: { second: '', standard: '', top: '' }

## 电梯显示规则

| 形态 | 显示方式 |
|------|---------|
| 独栋 | `{客梯数}客{货梯数}货`（分栋 `unitArea < 1500` 时只显示 `1货`） |
| 规则双拼/三拼（各单元面积段相同） | `分单元{客梯数}客{货梯数}货` |
| 不规则双拼/三拼，各单元电梯配置相同 | `分单元{客梯数}客{货梯数}货` |
| 不规则双拼/三拼，各单元电梯配置不同 | 按面积段从小到大用 `/` 连接各单元配置，并去重 |

## 备注

- 当前版本算法推导全程按独栋，所以 `unitArea = unitCap`，`form = '独栋'`
- 未来如需双拼转换，再调整 `unitArea` 和 `form`


## 双拼转换规则（最终输出阶段）

当前版本算法推导阶段全部按独栋处理。最终输出给用户时，才把需要拼合的独栋转换成双拼/三拼形式。

### 适用范围

| 产品类型 | 面积段 | 处理方式 |
|---------|--------|---------|
| 分层厂房 | ≤ 800 | **强制双拼**，不允许以独栋输出 |
| 分层厂房 | > 800 | 按奇偶处理 |
| 分栋厂房 | 所有 | 按奇偶处理 |
| 轻钢厂房 | 所有 | 按奇偶处理 |

### 基础转换

#### A. 分层厂房且面积段 ≤ 800：强制双拼
- `count` 为偶数：`count/2` 个 `area+area` 双拼
- `count` 为奇数：`(count-1)/2` 个 `area+area` 双拼，余 1 个 area 进入后续余数处理

#### B. 分层厂房 > 800、分栋厂房、轻钢厂房：按奇偶处理
- `count` 为偶数：`count/2` 个 `area+area` 双拼
- `count` 为奇数：`floor(count/2)` 个 `area+area` 双拼 + 1 个 area 独栋

### 处理分层厂房 ≤800 面积段的余 1

**只对分层厂房 ≤800 面积段产生的余 1 栋进行处理。**

1. **分层厂房 ≤800 面积段内部互相配对**
   - 收集所有分层厂房 ≤800 面积段中奇数余下的 1 栋
   - 两两配对，组成不规则双拼（如 600+800）

2. **仍余 1 个时，找分层厂房 >800 面积段的余 1 独栋配对**
   - 仅允许在分层厂房内部配对，不允许与分栋厂房或轻钢厂房混拼
   - 将落单的 ≤800 分层余 1 栋与 >800 分层面积段的余 1 独栋组成不规则双拼

3. **仍无法配对时，拆最小双拼组成三拼**
   - 从分层厂房 ≤800 面积段中找一个面积最小且 `count >= 2` 的面积段
   - 拆散其一个双拼，加入落单的 1 栋，组成非常规三拼（如 600+600+600）

### 输出展示
- 同面积段双拼：`area+area`
- 不规则双拼：`area1+area2`
- 非常规三拼：`area1+area2+area3`
- 独栋：`area`
