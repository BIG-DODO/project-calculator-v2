# 指标/测算总表下载（总表集成）需求文档

## 一、目标

将「三表联动」「投资估算」「静态投资分析」「动态投资分析」四大板块集成到一个完整的 Excel 工作簿中，形成以产品配置为核心的全流程测算总表。用户下载后可在本地修改关键参数，相关模块通过公式联动更新。

## 二、输出形式

- **页面入口**：导航栏「成果输出」→「指标/测算总表下载」。
- **下载文件**：`指标测算总表.xlsx`。
- **工作簿结构**：共 4 个部分，多个 sheet。

## 三、工作簿结构

### 第一部分：三表联动（5 个 sheet）

完全复制「规划/产品总表下载」中的 5 个 sheet：

1. **规划指标输入**
2. **产品配置选择**
3. **指标估算**
4. **产品配置详表**
5. **总体经济技术指标**

联动规则与 `MASTER_TABLE_LINKAGE.md` 完全一致。

### 第二部分：投资估算（4 个 sheet）

1. **规划指标**
   - 复刻投资估算模块中的「规划指标」sheet。
   - 包含：用地面积、容积率、计容/地上/地下/总建面、建筑密度、绿地率、道路面积、配套占比、研发办公占比、各产品建筑面积、产业大厦高度、分栋/分层独栋与双拼面积等。
   - 为投资估算完整版提供工程量引用源。

2. **加权平均造价表**
   - 复刻投资估算模块中的「加权平均造价表」。
   - 按产品类型（轻钢、分栋、分层、产业大厦）及其形态列出面积、单价、成本、加权平均单价。
   - 用于投资估算完整版中「单体建安成本」单价的公式引用。

3. **投资估算完整版**
   - 复刻投资估算模块的完整版 Excel。
   - 包含：土地配套费用、前期费用、建安工程成本（基础设施、景观、公建配套、单体建安）、开发间接费、营销费用、公司管理费、财务费用、发展成本合计。
   - 各小项工程量引用「规划指标」sheet；单体建安单价引用「加权平均造价表」。
   - 增值税公式结构：
     - 土地配套增值税：E = `(F8+F9)/1.06`，F = `D*E`
     - 前期费用增值税：E = `ROUND(SUM(F14:F21),2)/1.06`，F = `D*E`
     - 建安增值税：D = `0.09`，E = `ROUND(建安小计,2)/1.09`，F = `D*E`
   - 所有小计/合计 `SUM` 公式统一包 `ROUND(...,2)`。
   - 用户可编辑的输入项：土地出让金单价、大市政配套费单价、各产品单方造价、融资利率、融资占比、开发期数、单期开发周期等。

4. **投资估算简化版**
   - 由完整版汇总反推。
   - 展示：土地配套费用、前期费用、建安工程成本、开发间接费、营销费用、公司管理费、财务费用、发展成本合计。
   - 金额引用完整版对应小计行，占比公式包 `ROUND(...,2)`。

### 第三部分：静态投资分析（5 个 sheet）

1. **销售测算**
   - 包含：规划指标、建造成本、可售面积、加权平均售价、销售收入、土地成本、建安成本、税金及附加、土地增值税、营销费用、管理费用、财务费用、利润总额、所得税、净利润、销售净利率。
   - 销售收入公式引用「租售面积分配」销售收入合计，而非直接 `可售面积 × 加权平均售价`，以保证与明细一致。
   - 土地增值税引用「土地增值税测算表」应缴税额。

2. **租赁测算**
   - 包含：规划指标、建造成本、可租面积、加权平均租金、月租金、年租金收入、出租率、有效年租金、税金及附加、房产税、土地使用税、运营费用、净租赁收入、净租赁收入总额、租赁总投、NOI。
   - 年租金收入口径统一为「日租金 × 360」。
   - 有效年租金 = 年租金收入 × 出租率 / 100。
   - 净租赁收入总额 = 净租赁收入 per sqm × 可租面积 / 10000。

3. **租售面积分配**
   - 各产品类型销售面积/租赁面积、单价、加权平均价格。
   - 销售分配优先级：轻钢 → 分栋 → 分层 → 产业大厦。
   - 租赁分配优先级：产业大厦 → 分层厂房 → 分栋厂房 → 轻钢厂房；配套宿舍强制全部出租。
   - 加权平均售价/租金公式保留 6 位小数精度，显示仍为两位。

4. **土地增值税测算表**
   - 四级超率累进测算：转让收入、扣除项目、增值额、适用税率、速算扣除系数、应缴税额、土增税负率、单位面积土增税。
   - 所有中间结果和最终结果包 `ROUND(...,2)`。

5. **综合汇总**
   - 总投资、销售净利润、租赁年净收入、资金盈余/缺口、销售净利润覆盖租赁总投比例、销售净利率、租赁 NOI、总投资收益率、静态投资回收期。
   - 资金盈余/缺口 = 总投资 - (1-融资占比)*总投资 - 销售收入。
   - 静态投资回收期 = 总投资 / 租赁年净收入。

### 第四部分：动态投资分析（占位，3 个 sheet）

> 动态投资分析模块尚未开发完成，总表集成中先预留 sheet 结构，待开发后填充公式。

1. **多年现金流表**
   - 按年展示建设期和运营期现金流。
   - 引用静态分析中的销售/租赁收入、投资估算中的总投资和财务费用。

2. **敏感性分析**
   - 销售单价、租金单价、土地价格、建安成本、去化周期、出租率、运营费用率、建设周期、融资利率、融资占比等变量的 ±10%、±20% 波动对 NPV/IRR/自有资金 IRR/动态回收期的影响。

3. **关键指标汇总**
   - NPV、IRR、自有资金 IRR、动态投资回收期等核心指标。

## 四、跨部分联动规则

| 被引用部分 | 引用部分 | 联动关系 |
|---|---|---|
| 产品配置详表（Sheet4） | 总体经济技术指标（Sheet5） | Sheet5 引用 Sheet4 |
| 总体经济技术指标（Sheet5） | 投资估算、静态投资分析 | 投资/静态引用总建面、地上/地下建面、容积率、建筑密度、各产品面积等 |
| 规划指标（投资估算内 sheet） | 投资估算完整版 | 工程量引用 |
| 加权平均造价表（投资估算内 sheet） | 投资估算完整版 | 单体建安单价引用 |
| 投资估算 | 静态投资分析、动态投资分析 | 静态/动态引用总投资、综合单方成本、财务费用 |
| 静态投资分析 | 动态投资分析 | 动态引用销售收入、租赁收入、税金、NOI 等 |
| 指标估算（Sheet3） | 无反向联动 | 指标估算仅与规划指标输入联动，不参与后续财务测算反向传递 |

## 五、前置校验规则

下载「指标/测算总表」前需校验以下状态：

| 状态 | 提示信息 |
|---|---|
| 未进行投资估算 | 「请先完成投资估算」 |
| 已完成投资估算，但未完成静态投资分析 | 「请先进行静态投资分析」 |
| 已完成静态投资分析，动态投资分析未完成 | 可下载，动态分析 sheet 为占位空表；页面提示「动态分析未完成，总表中动态测算为占位」 |

注：「完成投资估算/静态投资分析」的定义为：用户进入对应页面并点击过「计算」按钮，结果已生成并存储在 `window.investmentEstimateResult` / `window.staticAnalysisResult` 中。

## 六、样式要求

- 统一使用微软雅黑字体。
- 不同部分用 sheet 标签颜色或标题区分。
- 表头：深蓝底（#0B2B5C）白字，加粗。
- 数据区：细边框，数字右对齐，保留两位小数。
- 关键指标/合计行：浅蓝底（#E0F2FE）或深蓝底白字突出。
- 所有数值尽量使用 Excel 公式，支持本地修改后联动。
- 所有求和公式统一包 `ROUND(SUM(...),2)`，中间计算结果也包 `ROUND(...,2)`，避免启用编辑后尾差或 `#VALUE!`。

## 七、数据来源

| 部分 | 数据来源 |
|---|---|
| 三表联动 | `projectData` + `window.configResult` |
| 投资估算 | `window.investmentEstimateResult` + `window.configResult` + `projectData` |
| 静态投资分析 | `window.staticAnalysisResult` + `window.investmentEstimateResult` |
| 动态投资分析 | `window.dynamicAnalysisResult` + 静态/投资估算结果（开发中） |

## 八、与「规划/产品总表下载」的关系

- 「规划/产品总表下载」是「指标/测算总表下载」的子集，仅包含前 5 个 sheet。
- 两者代码可复用：先实现三表联动的 5 个 sheet 生成逻辑，再在总表集成中追加投资估算、静态、动态 sheet。

## 九、实现顺序建议

1. 先实现「规划/产品总表下载」的 5 个 sheet；
2. 再实现「指标/测算总表下载」，复用前 5 个 sheet 并追加投资估算（规划指标、加权平均造价表、完整版、简化版）、静态分析（销售测算、租赁测算、租售面积分配、土地增值税测算表、综合汇总）、动态分析占位 sheet；
3. 最后统一测试跨 sheet 公式联动，重点验证启用编辑后无 `#VALUE!`、无尾差。
