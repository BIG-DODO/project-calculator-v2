## 1. 项目背景

本项目是一个基于Web的规划指标计算工具，根据用户输入的用地条件（用地面积、容积率、限高、配套占比等），自动计算各类厂房和配套建筑的产品配置。

## 2. 目标

根据用户输入的规划用地指标，自动生成产品配置表，包括：
- 各类厂房（轻钢、分栋、分层）的栋数、面积段、层数
- 配套建筑（产业大厦、配套楼、配套宿舍）的配置
- 容积率、建筑密度等指标的校验

**核心要求**：
- 容积率偏差 ≤ 0.5%（理想 ≤ 0.01%）
- 建筑密度偏差 ≤ 5%
- 面积段分布均衡（避免某面积段占比超过其他总和的2倍）

## 3. 全局变量定义

| 变量 | 含义 | 计算方式 |
|------|------|---------|
| S | 规划用地面积 | 用户输入 |
| F | 容积率 | 用户输入 |
| D | 建筑密度 | F<1.5→45%, 1.5≤F<2.0→42%, F≥2.0→40% |
| R1 | 科研办公用房占比 | 用户输入 |
| R2 | 配套用房占比 | 用户输入 |
| Hl | 限高 | 用户输入 |
| targetBase | 目标基底面积 | S × D |
| targetCap | 目标计容面积 | S × F |
| targetRdArea | 研发办公目标计容面积 | targetCap × R1 |
| targetAncillaryArea | 配套用房目标计容面积 | targetCap × R2 |
| totalAncillary | 配套用房目标计容面积（别名） | 同 targetAncillaryArea |
| exactBase | 精确基底面积（取整前） | 目标面积 / 层数 |

**术语定义**：
- **Bh** = 较高层数/效率的厂房类型
- **Bl** = 较低层数/效率的厂房类型
- **N** = 层数（用于高度公式，区别于容积率F）
- **火车头** = 轻钢厂房首层局部加高部分，用于办公或展示
- **双拼** = 两栋厂房合并建造，共享墙体。本版本算法推导阶段**不处理双拼**，仅在最终输出阶段可选项转换
- **FL预处理** = 固定部分厂房配置，使剩余层数类型 ≤ 2
- **强排** = 强制性规划指标排列，即在满足规划条件的前提下进行产品配置

## 5. 关键规则

### 5.1 算法推导核心原则

**算法推导全程全部按独栋处理，不考虑双拼转换。**

- 轻钢厂房：`base = area`，`unitCap = base × 2 + 200`
- 分栋厂房：`base = area`，`unitCap = base × N`
- 分层厂房：`base = area`，`unitCap = base × N`
- 双拼转换仅在最终输出阶段处理，不属于算法推导逻辑

### 5.2 分层厂房限高处理

- 当分层厂房高度超过限高时，**自动逐层减1**
- 允许降到任意层数（无6层下限）
- ≥5层：归类为分层，备注"因限高原因，分层降低层数"
- ≤4层：归类为分栋，面积段与原分栋并集去重（保持原始面积段，不转换双拼）
- 若降至1层仍超限高，报错并移除此层数选项

### 5.3 配套用房规则

**UI选择规则**：
- `totalAncillary ≤ 2400`：用户可任意选择（仅配套楼 / 仅配套宿舍 / 两者都选）
- `totalAncillary > 2400`：强制必须选配套宿舍，配套楼可选（不强制）

**配套楼面积边界**：
| 总面积 | 配套楼面积 | 配套宿舍 | 配套楼层数 |
|--------|-----------|---------|-----------|
| < 1200 | 全部（可<1200） | 无 | 2层 |
| 1200-2400 | 全部 | 无 | 3层 |
| 2400-3600 | 1200（固定） | 剩余 | 3层 |
| 3600+ | 用户选择面积段（钳制在 [1200, 2400]） | 剩余 | 3层 |

**配套楼取整规则（三次取整）**：
1. 基底 = floor(exactBase / 50) × 50
2. 若偏差 > 0.2%，基底 = floor(exactBase / 20) × 20
3. 若仍 > 0.2%，基底 = floor(exactBase / 10) × 10

**配套宿舍**：
- 初始基底范围：**600-1200**
- 基底取整：`floor(exactBase / 20) × 20`
- 最终基底必须钳制在 **600-1200** 之间
- 高度超限时两次调整：标准层高3.6m → 降低至3.3m → 仍超限则用上限反推层数
- 强制层数 ≥ 2层

### 5.4 产业大厦

- 面积段由 UI 单选确定
- 层数 = max(3, ceil(targetRdArea / areaRef))
- 默认标准层层高 **4.5m**，顶层同标准层
- 高度公式：`6.6 + (N-1) × 标准层 + 1.2`
- 产业大厦高度与分层厂房高度**无关**，仅以限高为约束

**限高超限处理（先降层高、再减层数）**：

1. 按标准层 4.5m 计算建筑高度
2. 若超限：标准层降至 **4.2m**，顶层同步降至 4.2m，层数保持不变
3. 若仍超限：恢复标准层 4.5m，通过限高反推最大层数
   - `最大层数 = max(3, floor((Hl - 6.6 - 1.2) / 4.5) + 1)`
4. 若反推后（至少3层）仍超限：报错"限高不足，无法放置3层产业大厦"

**面积段取整**：先 50 倍数，偏差>0.2% 则 20 倍数

**输出字段**：除 base/floors/totalHeight 外，增加 `standardHeight` 字段记录最终标准层层高（4.5 或 4.2），供前端展示

### 5.5 Bh/Bl核心规则

- **Bh** = 效率较高的厂房类型
- **Bl** = 效率较低的厂房类型
- 平均效率 ≥ Bh效率 → 取消Bl，只用Bh（不考虑密度，仅用容积率控制）
- 平均效率 ≤ Bl效率 → 取消Bh，只用Bl（不考虑密度，仅用容积率控制）
- 平均效率介于两者之间 → 混合分配

**3.5层处理**：
- 3.5 vs 3 → 3.5是Bh
- 3.5 vs 4 → 3.5是Bl

**轻钢等效层数**：2.1

### 5.6 单一层数规则

- 当某一场景退化为单一厂房层数时，只用容积率控制，不考虑密度
- 分布不均时先触发丰富面积段，仍不满足则用手动分配算法

### 5.7 FL预处理

- 目标：将多种层数类型固化为最多2种
- 触发条件：`splitFloors.length + layerFloors.length > 2`
- 处理顺序：优先处理分层，其次分栋
- 计算：
  - 取当前最高层数 `maxFloor` 和最大面积段 `maxArea`
  - `a = maxArea × maxFloor`
  - 分层：`FL = targetCap / a / 2`
  - 分栋：`FL = targetCap / a`
- 固定栋数：`fixCount = max(1, floor(FL))`
- 固定配置：最大面积段 × maxFloor，count = fixCount
- 固定后移除该层数，循环直到层数类型 ≤ 2

### 5.8 三种厂房类型同时存在

- 强制分配1栋最低层数、最小面积的分层厂房
- 剩余部分退化为分栋 + 轻钢（不再分配其他分层）
- 若分栋仍有多种层数，按 FL 预处理规则继续固化

### 5.9 丰富面积段

- 触发条件：遍历算法后最优解仍无法满足容积率允许误差
- 增加上限（按**用地面积S**判断）：≤20000m²→+1，≤50000m²→+2，>50000m²→+3
- 方法：递减（x-50, x+50, x-100）+ 内插（相邻两值的中间值）
- 取整：所有新增值必须是50的倍数
- 去重：新增值不能与已有值重复
- 丰富面积段基于原始面积段（独栋），不转换双拼

### 5.10 手动分配算法

当某厂房类型出现分布不均（某面积段栋数 > 其他总和 × 2 且总栋数 ≥ 5）时触发：

1. 计算参考总栋数：`refCount = ceil(S_type × 2 / (a + b))`，其中 S_type 为该类型实际占地，a 为最小面积段，b 为最大面积段
2. **极端情况**：若 `S_type < b`，输出 1 栋，面积段 = ceil(S_type / 10) × 10
3. **`refCount ≤ 3`**：统一面积段，面积段 = ceil(S_type / refCount / 10) × 10
4. **`3 < refCount ≤ 6`**：
   - aa = max(1, ceil(refCount × 2 / 3) - 2)
   - bb = 1
   - cc = refCount - aa - bb
   - 若 cc ≤ 0，退化为统一面积段
   - 中间面积段 c = ceil((S_type - a×aa - b×bb) / cc / 10) × 10
   - c 必须钳制在 **[a, b]** 区间内
   - 输出：aa 栋 a + cc 栋 c + bb 栋 b
5. **`refCount > 6`**：
   - aa = max(1, ceil(refCount × 2 / 3) - 2)
   - bb = 2
   - 中间面积段 c 独立计算
   - c 必须钳制在 **[a, b]** 区间内
   - 输出：aa 栋 a + cc 栋 c + bb 栋 b

### 5.11 双拼转换（最终输出阶段，非算法推导）

> 注：本版本算法推导阶段全部按独栋处理。双拼转换仅作为最终输出的可选后处理，具体规则待算法跑通后确定。

2. **分层厂房层数限制**：最多只能选择 **2种** 层数
3. **产业大厦面积段**：只能选择一个面积段

### 7.1 期望输出验证标准

每个场景的算法输出必须同时满足以下三个条件：

**1. 容积率偏差**
- 允许误差：≤ 0.5%
- 理想误差：≤ 0.01%
- 计算公式：`|实际计容面积 - 目标计容面积| / 目标计容面积`

**2. 建筑密度偏差**
- 允许误差：≤ 5%
- 计算公式：`|实际基底面积 - 目标基底面积| / 目标基底面积`

**3. 栋数分布均衡性**
- 同一厂房类型中，当该类型的总栋数 ≥ 5 时：
  - 任意单一面积段的栋数 ≤ 其他面积段栋数总和 × 2
- 若触发此条件，需通过丰富面积段或手动分配算法优化

## 8. 异常处理

### 8.1 输入参数异常
- `landArea <= 0`：报错"用地面积必须大于0"
- `far <= 0`：报错"容积率必须大于0"
- `heightLimit <= 0`：报错"限高必须大于0"
- `ancillaryRatio < 0` 或 `rdRatio < 0`：报错"占比不能为负"
- `ancillaryRatio + rdRatio > 1`：报错"配套占比+科研占比不能超过1"

### 8.2 产品选择异常
- 选择分层+轻钢但未选择分栋：报错"未选择分栋厂房"
- 分栋或分层层数类型超过2种：报错"层数类型超过2种"
- R2>0但未选配套楼/配套宿舍：报错"配套用房未选择"
- 配套面积>2400且只选配套楼：报错"配套用房面积未用尽，请勾选配套宿舍"

### 8.3 产品配置异常
- 某层数因限高完全不可用：记录原因，从 selectedProducts 中移除
- 分层降至1层仍超限高：报错"分层厂房即使1层仍超过限高"
- 固定产品超过密度/容积率上限：尝试移除 FL 固定配置，仍超限则报错。**不允许减少产业大厦面积作为兜底策略**
- 所有面积段都不满足容积率：触发丰富面积段，仍不满足则用手动分配
- 密度超限（>5%）：回退到上一轮 counts，或放宽密度约束

### 8.4 兜底策略
- 当所有算法都无法满足要求时，输出最接近的可行解，并标注偏差
- 提供手动调整入口，允许用户手动修改配置
