## 一、已修补问题

### 1. 【关键】三种厂房类型处理后未清空剩余分层楼层
**问题描述**：
步骤三 3.4 节处理三种厂房类型同时存在的情况时，强制分配 1 栋最低层数最小面积的分层后，仅移除了 `minLayerFloor`，未清空剩余的 `layerFloors`。这会导致剩余分层楼层仍进入 Bh/Bl 核心算法，违反"剩余部分退化为分栋 + 轻钢，不再分配其他分层"的规则。

**修复方式**：
在移除 `minLayerFloor` 后增加 `layerFloors = []`，确保全部分层仅保留强制分配的 1 栋。

```
layerFloors = 过滤掉 minLayerFloor
layerFloors = []  // 强制分配的这1栋分层即为全部分层
```

### 2. 【中危】`enrichConfigs` 使用未传入的变量 D
**问题描述**：
`enrichConfigs` 函数体内使用 `landArea * D` 计算 `totalTargetBase`，但函数签名仅为 `(configs, landArea)`，D 不在作用域内。

**修复方式**：
- 函数签名改为 `(configs, landArea, D)`
- 调用处从 `targetBase / landArea` 计算 D 后传入

---

### 3. 【中危】`manualDistribute` 中间面积段 c 可能越界
**问题描述**：
`manualDistribute` 中计算出的中间面积段 `c` 理论上应在 `[a, b]` 之间，但由于 `S_type` 取整和 `refCount` 估算的偏差，极端情况下 `c` 可能小于 `a` 或大于 `b`，破坏"a 为最小、b 为最大"的语义。

**修复方式**：
两处 `c` 的计算后增加钳制：
```
c = ceil(remaining / cc / 10) * 10
c = max(a, min(b, c))
```

### 4. 【低危】step0 中 1 层降级为分栋的产品逻辑
**描述**：
当分层厂房降至 1 层仍满足限高时，会被归类为分栋。1 层分栋厂房高度约 9m，产品逻辑上较奇怪，但规则允许"降到任意层数"。

**处理**：保留当前逻辑，作为边界情况允许。

---

### 5. 【低危】`calc_dorm` 最终 base 钳制后的面积偏差
**描述**：
当 `finalBase` 被钳制到 600 或 1200 时，`unitArea = finalBase * floors` 可能与 `dormCap` 有较大偏差。当前规则可接受此偏差。

**处理**：保留当前逻辑，作为已知限制。
