## 一、用户澄清后的高度计算规则

**核心规则**：
- 所有产品建筑高度 = Excel 表格公式 + 1.2m 女儿墙/屋顶高度
- 唯一例外：**轻钢厂房**表格公式已直接写为 `12+1.2=13.2`，无需再加
- **3.5层分栋厂房**：建筑高度按 4层计算，即 **22.5m**（已含女儿墙）

## 三、已修复问题

### 问题1：产业大厦高度公式缺少女儿墙 1.2m
**修复前**：`6.6 + 4.5 * (N - 1)`（误删了 +1.2）  
**修复后**：`6.6 + 4.5 * (N - 1) + 1.2`

### 问题2：配套宿舍高度公式楼层数错误
**修复前**：`4.8 + 3.6 * (N - 2) + 1.2`  
**修复后**：`4.8 + 3.6 * (N - 1) + 1.2`

### 问题3：3.5层高度曾一度被改为 22.5m 未含女儿墙
**说明**：4层含女儿墙高度 = 22.5m，因此 3.5层 = 22.5m 已含女儿墙，无需再加 1.2m。

## 五、`computeTotalHeight` 最终版本

```
function computeTotalHeight(type, floor):
   if type == 'light-steel':
      return 13.2
   else if floor == 3.5:
      // 3.5层建筑高度按4层计算（22.5m，已含女儿墙）
      return 22.5
   else if type == 'tower':
      return 6.6 + 4.5 * (floor - 1) + 1.2
   else if type == 'dorm':
      return 4.8 + 3.6 * (floor - 1) + 1.2
   else:
      // split / layer / support（N>=3 时）
      return 7.2 + 5.1 + 4.5 * (floor - 2) + 1.2
```

## 八、2026-07-09 补充：产业大厦"先降层高、再减层数"规则

### 新增规则
用户要求产业大厦超限高时，优先降低标准层层高（4.5m → 4.2m），仍超限再减层数，类似配套宿舍逻辑。

### 处理流程
1. 初始层数 = max(3, ceil(targetRdArea / areaRef))
2. 按标准层 4.5m 计算高度：`6.6 + 4.5 * (N-1) + 1.2`
3. 超限 → 标准层降至 4.2m，高度 = `6.6 + 4.2 * (N-1) + 1.2`
4. 仍超限 → 恢复 4.5m，反推层数 = `max(3, floor((Hl - 6.6 - 1.2) / 4.5) + 1)`
5. 反推后仍超限 → 报错
