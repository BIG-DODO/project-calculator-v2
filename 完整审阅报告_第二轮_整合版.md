## 第二部分：本轮新发现的严重问题（5项）

### 🔴 S1: 双拼转换条件永假 — 分层厂房被重复双拼

**位置**: ALGORITHM(1).md 3.6节 convertToDuplex 第498行

**问题描述**:
- createConfigs中分层<=800时 `base = area * 2`（600->1200, 800->1600）
- convertToDuplex中判断条件 `c.base <= 800`，但强制双拼后base最小为1200，条件**永远为false**
- 导致"分层<=800跳过转换"的保护完全失效，所有分层厂房会被**再次双拼**
- 例：分层600面积段8层，算法输出4栋 -> 实际基底=1200 -> convertToDuplex误判为原生1200 -> 偶数4栋转为2栋2400双拼 -> 最终显示2400面积段（完全错误）

**用户澄清+最终方案**:

为避免上述问题，采用**前置转换方案**：

1. **算法入口前置处理**：对用户选择的分层面积段中<=800的，先转换为双拼后的面积段（600->1200, 800->1600），去重后作为算法的初始配置
2. **保留原始面积映射**：在config中增加 `originalArea` 字段，记录转换前的原始面积段
3. **反向映射输出**：convertToDuplex按 `originalArea` 做反向映射，确保最终输出正确

**具体算法**:

```javascript
// === 前置处理（算法入口）===
function preprocessLayerAreas(originalAreas):
   duplexSourceMap = {}   // 转换后面积 -> 原始面积
   processedAreas = []
   
   for area in originalAreas:
      if area <= 800:
         convertedArea = area * 2
         duplexSourceMap[convertedArea] = area
         if convertedArea not in processedAreas:
            processedAreas.push(convertedArea)
      else:
         if area not in processedAreas:
            processedAreas.push(area)
   
   return {
      areas: processedAreas.sort((a,b)=>a-b),
      duplexSourceMap: duplexSourceMap
   }

// 例：用户选 [600, 800, 1000, 1200]
// 处理后：areas = [1000, 1200, 1600], duplexSourceMap = {1200: 600, 1600: 800}
```

```javascript
// === createConfigs中保留原始面积信息 ===
configs.push({
   id: type, type: typeName, base, unitCap,
   originalArea: area,           // 保留原始面积段
   duplexSource: (type == 'layer' && area <= 800) ? area : null,
   floors: floor, totalHeight: totalH
})
```

```javascript
// === convertToDuplex中按原始面积做反向映射 ===
function convertToDuplex(counts, configs, duplexSourceMap):
   output = []
   
   for i from 0 to configs.length - 1:
      c = configs[i]
      count = counts[i]
      if count <= 0: continue
      
      sourceArea = duplexSourceMap[c.base] || null
      
      // 情况1：分层且来自<=800的强制双拼（1200->600, 1600->800）
      if c.id == 'layer' and sourceArea != null:
         if count % 2 == 0:
            output.push({
               type: c.type,
               displayBase: sourceArea,        // 显示600或800
               actualBase: c.base,              // 实际基底1200或1600
               floors: c.floors,
               duplexCount: count / 2,
               isDuplex: true,
               note: sourceArea + '+' + sourceArea + '双拼'
            })
         else:
            duplexCount = floor(count / 2)
            if duplexCount > 0:
               output.push({
                  type: c.type,
                  displayBase: sourceArea,
                  actualBase: c.base,
                  floors: c.floors,
                  duplexCount: duplexCount,
                  isDuplex: true,
                  note: sourceArea + '+' + sourceArea + '双拼'
               })
            output.push({
               type: c.type,
               displayBase: sourceArea,
               actualBase: sourceArea,
               floors: c.floors,
               count: 1,
               isDuplex: false,
               note: sourceArea + '独栋'
            })
      
      // 情况2：分层原生>=1000面积段（1000, 1200原生）
      else if c.id == 'layer':
         if count % 2 == 0:
            output.push({
               type: c.type,
               base: c.base * 2,
               floors: c.floors,
               count: count / 2,
               isDuplex: true,
               note: '双拼'
            })
         else:
            duplexCount = floor(count / 2)
            if duplexCount > 0:
               output.push({
                  type: c.type,
                  base: c.base * 2,
                  floors: c.floors,
                  count: duplexCount,
                  isDuplex: true,
                  note: '双拼'
               })
            output.push({
               type: c.type,
               base: c.base,
               floors: c.floors,
               count: 1,
               isDuplex: false,
               note: '独栋'
            })
      
      // 情况3：非分层类型（轻钢、分栋）—— 原有逻辑不变
      else:
         // ... 原有convertToDuplex逻辑
         
   return output
```

**关键规则**:
| 算法中的base | 来源 | duplexSourceMap | 最终显示 |
|-------------|------|----------------|---------|
| 1200 | 600转换 | 600 | 600+600双拼 |
| 1600 | 800转换 | 800 | 800+800双拼 |
| 1000 | 原生 | null | 1000+1000双拼（偶数）或1000独栋（奇数余1） |
| 1200 | 原生 | null | 1200+1200双拼（偶数）或1200独栋（奇数余1） |

### 🔴 S2: 振荡检测完全失效 — 已知问题#1仍未修复

**位置**: ALGORITHM(1).md 5.3节

**问题描述**:
- `oscillationHistory = []` 在5.3节函数开头初始化，每次调用都重置
- `detectOscillation()` 检查 length >= 3 才判定振荡，但重置后长度永远是0
- 且代码逻辑是**先检测后记录**，检测时永远无历史数据
- 这意味着已知问题#1（场景4密度偏差17.5%）的修复方案实际上不会生效

**修复方案**:

```javascript
// 在runOuterLoop函数外层（循环前）初始化
oscillationHistory = []

function detectOscillation():
   if oscillationHistory.length < 3: return false
   last3 = oscillationHistory.slice(-3)
   return (last3[0] > 0 && last3[1] < 0 && last3[2] > 0) ||
          (last3[0] < 0 && last3[1] > 0 && last3[2] < 0)

// 在runOuterLoop循环中：
// 1. 先记录本轮偏差
// 2. 再检测是否振荡
// 3. 再根据检测结果决定下一步

densityDiff = totals.base - targetBase
if densityDiff > targetBase * 0.05: oscillationHistory.push(1)
else if densityDiff < -targetBase * 0.05: oscillationHistory.push(-1)
else: oscillationHistory.push(0)
if oscillationHistory.length > 5: oscillationHistory.shift()

// 然后检测振荡
distCheck = checkDistribution(...)
if distCheck.needsOptimization or detectOscillation():
   // 执行优化/手动分配
```

### 🔴 S5: 分栋3.5层高度

**用户澄清**: 3.5层的建筑高度等于4层高度（23.7m），因为第四层只做一半面积，另一半是屋顶露台。屋面高度与4层相同。

**结论**: 算法中23.7m是正确的，**不是bug**。

**建议补充文档说明**（需求文档第4节分栋厂房行）:
```
分栋厂房 3.5层：高度=23.7m（等同于4层），第4层为半层+屋顶露台
```
